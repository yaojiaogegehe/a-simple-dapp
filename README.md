# a-simple-dapp
一个python运行的元气骑士游戏，它包含下面的规则

请帮我设计一款原创的 2D 像素风顶视角 Roguelike 双摇杆射击游戏，用 Unity2D 实现。整体风格和玩法参考手机端的地牢闯关射击游戏，但不要照搬任何现有游戏的名称、角色或关卡，只保留类型特点和系统结构。

需要的核心特征如下

一、基础定位
1 游戏类型为俯视视角 Roguelike Dungeon Crawler 与 Twin Stick Shooter 结合
2 操作方式为移动摇杆加射击摇杆，支持键鼠和手柄双方案设计
3 美术为像素风 2D，画面节奏偏快，战斗打击感强，子弹量较多但仍可读性清晰

二、核心玩法循环
1 玩家从主大厅或据点出发，选择角色与初始武器，进入随机生成的地牢关卡
2 每次游玩为一局 Run，地牢由若干章节组成，每章内包含多个房间，房间布局、怪物组合、掉落物和事件为程序化随机生成
3 玩家在房间内进行移动、射击、闪避，击败敌人，清理房间后可以选择下一间房或前往商店房、宝箱房、事件房等
4 通过战斗和探索获得新武器、装备、消耗品与被动强化，构成一局内的 Build 搭配
5 击败每章末尾 Boss 进入下一章节，整局通关后回到主大厅
6 角色死亡会结束本局，局内获得的大部分道具清空，只保留部分长期成长养成资源，用于解锁新角色、武器或被动加成

三、游戏模式设计
1 普通闯关模式
请设计一个标准的主线闯关模式，包含不少于三章地牢，每一章有不同主题的场景与怪物类型，例如地牢、森林、科技基地等。每章结束有一个 Boss 战，Boss 需要具备阶段变化、弹幕模式变化和技能循环。
2 高难度模式
在普通模式基础上增加一档高难度模式，提升敌人攻击力、血量、移动速度和弹幕密度，同时提高奖励获取效率。可以加入额外精英怪、陷阱或负面环境效果，让这一模式服务于高玩与刷资源玩家。
3 Boss 连战模式
设计一个只包含 Boss 战的模式，按顺序或随机连续挑战多个 Boss。中间可以设置简短的补给阶段，用于更换武器或购买消耗品。Boss 连战模式的目标是考验玩家对机制的熟悉程度与角色配装能力。
4 无限或循环模式
设计一个可以无限推进的模式，敌人和房间会循环出现，但每一轮敌人强度上升。加入简单的积分或波数记录系统，用于记录玩家最高坚持时长或最高波数。这个模式主要用于刷资源和追求极限挑战。

四、角色与武器系统
1 设计不少于三名可玩角色，角色的差异体现在主动技能、被动能力、初始武器或基础属性等方面，例如某个角色擅长近战，另一个角色擅长远程持续输出，还有一个角色偏向控制或辅助。
2 游戏中需要有数量较多的武器品类，包含手枪、步枪、散弹、激光、近战武器、投掷类武器等。每种武器应有明确的攻击方式、射击节奏、弹道特征和资源消耗方式。
3 武器与角色之间存在一定的搭配关系，某些被动效果或天赋可以加强特定类型武器的表现，鼓励玩家在一次 Run 中尝试不同构筑路线。

五、关卡与敌人设计
1 地牢房间采用 Tilemap 形式构建，包含普通战斗房、精英房、商店房、宝箱房、事件房和 Boss 房等不同功能类型。
2 敌人种类应覆盖近战冲锋型、远程射击型、范围攻击型、召唤型等，各自拥有不同的移动 AI 与攻击模式。
3 Boss 需要具备清晰的招式循环，允许玩家通过多次游玩学习攻击模式并形成对策，而不是纯粹堆数值。

六、数值与成长
1 建立一个基础的数值框架，包括角色生命值、护盾、移动速度、武器伤害、攻速、暴击率、子弹数量等。
2 设置局外长期成长系统，例如通过收集资源解锁新角色、扩充武器池、提升初始属性上限等，使玩家在多次失败后仍有进步感。
3 控制数值膨胀，使游戏在中后期保持一定挑战性，而不是简单地无脑碾压。

七、表现与实现建议
1 所有系统设计需要适合在 Unity2D 中实现，建议使用刚体和碰撞体构建角色与子弹的物理交互，用 Animator 或状态机控制角色、敌人和特效动画，用 Tilemap 构建关卡并通过代码进行随机生成。
2 请把以上所有内容整理成一份较为详细的游戏设计方案结构，可以拆分为模块说明，如输入与控制系统、角色与武器系统、关卡与生成系统、敌人与 Boss 设计、模式与进程、数值与成长、UI 与交互等，方便我后续据此编写 Unity 项目代码。
3 整体要求是原创 IP，但玩法风格接近像素风 Roguelike 双摇杆射击手游，确保不侵犯任何现有商业游戏的美术、剧情或具体关卡设计。
